# CPU+NPU 异构SoC 学习路线

> 基于本项目的完整分析，这是一个非常优秀的学习材料——它涵盖了现代异构SoC设计的核心技术栈。
> 以下是按层次递进的详细学习路线。

---

## 第一阶段：基础协议与总线（第1-2周）

### 1.1 AXI4协议 — 整个SoC的"神经系统"

这个项目**处处都是AXI**，必须先吃透它。

**阅读顺序：**

1. 先读 `docs/DESIGN_REPORT.md` 中关于AXI协议的章节（有中文详细讲解）
2. 再读 `src/dma/inc/amba_axi.svh` 和 `amba_axi_pkg.sv` — 理解信号定义
3. 对照阅读 `src/axi_crossbar/axi_lite2axi.sv` 和 `axi2axi_lite.sv` — 理解AXI-Lite与AXI4的转换

**关键知识点：**

- 5个通道：AW（写地址）、W（写数据）、B（写响应）、AR（读地址）、R（读数据）
- 握手协议：`valid`/`ready` 信号的时序规则（`valid`不能等待`ready`）
- 突发传输：INCR、FIXED、WRAP三种模式的区别
- 4KB边界限制：为什么AXI规定burst不能跨越4KB边界
- ID标签：实现乱序完成的基础

**动手练习：**

```text
在testbench中找到DMA的AXI事务，用ModelSim波形观察：
1. 一次完整的读传输（AR→R通道握手序列）
2. 一次完整的写传输（AW→W→B通道握手序列）
3. 观察dma_axi_if.sv中的SVA断言如何检查协议合规
```

### 1.2 AXI Crossbar — 理解SoC互联

**阅读文件：** `src/axi_crossbar/axicb_crossbar_top.sv`

**关键知识点：**

- 4×4交叉矩阵如何实现多主多从并发访问
- 基于优先级的轮询仲裁（`axicb_round_robin_arbiter.sv`）
- 乱序完成管理（`axicb_slv_ooo.sv`）— 用per-ID FIFO跟踪事务
- Outstanding事务：为什么需要多个outstanding，如何提高总线利用率

**学习建议：**

画出这个SoC的完整地址映射表，理解CPU访问`0x4000_0000`时，信号如何经过crossbar到达DDR：

```text
CPU (PicoRV32)
  → AXI-Lite接口
    → axi_lite2axi (协议转换)
      → axicb_crossbar (slv0端口, ID mask 0x10)
        → 地址解码 → mst0 (DDR: 0x4000_0000)
                   → mst1 (NPU RAM: 0x0000_1000)
                   → mst2 (DMA CSR: 0x0002_1000)
                   → mst3 (NPU CSR: 0x0003_0000)
```

## 第二阶段：CPU子系统（第2-3周）

### 2.1 PicoRV32 RISC-V核心

**阅读文件：** `src/cpu/picorv32.v`（约2000行）

**关键知识点：**

- RV32I指令集：37条基础整数指令
- 2级流水线结构：取指→执行（非经典5级，更适合FPGA）
- 寄存器文件：32×32位双端口
- 移位器：2级移位（非桶形移位器），面积换速度的权衡
- 内存接口：原生AXI-Lite输出

**学习建议：**

1. 对照RISC-V规范（riscv.org）阅读`picorv32.v`中的指令解码逻辑
2. 重点关注`latched_store`、`latched_stalu`等信号——理解PicoRV32如何用最少的寄存器实现流水线
3. 思考：为什么这个设计没有启用MUL/DIV？（答案：NPU承担了所有计算密集任务）

### 2.2 CPU Wrapper与地址解码

**阅读文件：** `src/cpu/picorv32_axi.v`

**关键知识点：**

- 地址路由器：ROM（0x0000_0000）、RAM（0x1000_0000）、AXI外设的地址解码
- AXI-Lite适配器：将PicoRV32的内存接口转为标准AXI-Lite
- 本地存储器：4KB ROM + 4KB RAM的用途

### 2.3 CPU固件 — 理解软硬件协同

**阅读文件：** `src/instr_data.S`（仅100行，但极其关键）

这是CPU控制NPU的完整流程：

```asm
# 核心流程：
# 1. 配置DMA描述符（源地址=DDR图像数据，目的地址=NPU RAM）
# 2. 启动DMA传输
# 3. 轮询DMA完成
# 4. 触发NPU推理
# 5. 轮询NPU完成
# 6. 读取分类结果
```

**学习建议：**

逐行阅读这段汇编，理解：

- CPU如何通过MMIO（内存映射IO）访问DMA和NPU的CSR寄存器
- 轮询 vs 中断的权衡（这里用轮询，因为PicoRV32的IRQ被禁用了）
- 为什么DMA是必要的？（CPU直接搬数据太慢，且无法充分利用总线带宽）

## 第三阶段：DMA控制器（第3-4周）

### 3.1 DMA架构 — 7层层次化设计

**阅读顺序（由外到内）：**

```text
1. dma_axi_top.sv      — 信号打包层，理解端口定义
2. dma_axi_wrapper.sv   — CSR + 功能逻辑集成
3. dma_csr.sv           — CSR寄存器文件（重点！）
4. dma_fsm.sv           — 4状态控制器
5. dma_func_wrapper.sv  — 功能核心集成
6. dma_streamer.sv      — 读/写突发引擎
7. dma_axi_if.sv        — AXI4 Master接口引擎（最复杂）
8. dma_fifo.sv          — 数据缓冲FIFO
```

**关键知识点：**

#### 1. CSR寄存器映射（`dma_csr.sv`）

- 理解如何用AXI-Lite接口实现寄存器读写
- 2个独立描述符的设计意图（可以实现链式DMA或ping-pong缓冲）

#### 2. 状态机（`dma_fsm.sv`）

```text
IDLE → CFG（加载描述符）→ RUN（执行传输）→ DONE（完成/错误）
```

理解每个状态转换的条件

#### 3. 突发引擎（`dma_streamer.sv`）

- 读引擎：从源地址读数据到FIFO
- 写引擎：从FIFO写数据到目的地址
- 读写可以流水线化（读在进行时，写可以同时进行）

#### 4. 4KB边界处理（`dma_axi_if.sv`）

- 当传输跨越4KB边界时，自动拆分为多次burst
- 这是AXI协议的硬性要求，也是面试常考点

#### 5. SVA断言（`dma_axi_if.sv`）

- 学习如何用SystemVerilog Assertion验证AXI协议合规性
- 例如：`assert property (@(posedge clk) arvalid |-> !arready || ...)`

**动手练习：**

```text
在testbench中找到以下测试用例并运行：
Test 3: 基本DMA传输（burst length = 4）
Test 14: 4KB边界交叉传输
Test 20: 非对齐地址传输
Test 27: DMA abort
用波形观察dma_fsm的状态转换和dma_axi_if的5个通道信号
```

## 第四阶段：NPU — 神经网络加速器（第4-6周，最核心）

### 4.1 CNN基础（前置知识）

在深入RTL之前，先理解NPU要加速的计算：

```text
卷积操作: Output[n][m] = ΣΣ Input[n+i][m+j] × Kernel[i][j] + Bias
```

CIFAR-10网络结构：

```text
输入: 32×32×3 (RGB图像)
  → Conv1: 5×5, 3→32通道, pad=2 → 32×32×32
  → ReLU → MaxPool 2×2 → 16×16×32
  → Conv2: 5×5, 32→64通道, pad=2 → 16×16×64
  → ReLU → MaxPool 2×2 → 8×8×64
  → GAP (全局平均池化) → 64维向量
  → FC (64→10) → 10维向量
  → Argmax → 分类结果 (0-9)
```

### 4.2 脉动阵列 — NPU的计算核心

**阅读顺序：**

```text
1. pe.sv                          — 单个处理单元（最小颗粒）
2. mm_systolic_4x4.sv            — 4×4子阵列
3. mac_array_40x32_stream.sv     — 完整40×32阵列
4. docs/NPU_DESIGN_REPORT.md     — 数据通路详细分析
```

**关键知识点：**

#### 1. PE（处理单元）（`pe.sv`）

```text
psum_out = psum_in + W × A_in
```

- INT8乘法 + INT32累加
- 2级流水线：乘法阶段 + 累加阶段
- 权重驻留（Weight-Stationary）：权重加载一次，复用多次

#### 2. 4×4子阵列（`mm_systolic_4x4.sv`）

- 16个PE排列成4×4网格
- 数据流动方式：输入数据水平流动，部分和垂直流动
- 带有bias加法和ReLU激活

#### 3. 40×32完整阵列（`mac_array_40x32_stream.sv`）

- 80个4×4子阵列 = 1280个PE
- 为什么是40×32？与卷积核尺寸和通道数的关系
- Time-skew对齐：如何处理不同时间到达的数据

#### 4. 脉动阵列数据流（重点理解）

```text
时钟周期1: W[0][0]×A[0],   W[0][1]×A[0],   ...
时钟周期2: W[0][0]×A[1],   W[0][1]×A[1],   ... (同时累加周期1的结果)
...
```

理解"脉动"的含义：数据像心脏跳动一样有节奏地流过阵列

### 4.3 数据准备 — im2col变换

**阅读文件：** `src/npu/dmac_im2col_stream.sv`

**关键知识点：**

- im2col：将卷积操作转化为矩阵乘法
- 为什么需要im2col：脉动阵列本质上是矩阵乘法引擎
- 组合逻辑实现（零延迟）：纯组合电路完成地址映射

```text
原始卷积: 输入[32][32][3] × 卷积核[5][5][3][32] → 输出[32][32][32]
im2col后: 矩阵[1024][75] × 矩阵[75][32] → 矩阵[1024][32]
```

### 4.4 后处理单元

**MaxPool**（`src/npu/ppu_maxpool.sv`）：

- 流水线化2×2最大池化
- 与结果写回重叠执行（隐藏延迟）

**GAP + FC + Argmax**（`src/npu/gap_fc_logits.sv`）：

- 全局平均池化：8级流水线树形归约
- FC层：64→10全连接
- Argmax：比较器树找最大值索引

### 4.5 NPU控制与数据流

**阅读文件：** `src/npu/npu_top.sv` 和 `src/npu/conv_top.sv`

理解完整的推理流程：

```text
1. CPU通过CSR配置NPU参数
2. DMA将图像数据搬运到NPU本地RAM
3. NPU启动Conv1: im2col → 脉动阵列计算 → MaxPool
4. NPU启动Conv2: im2col → 脉动阵列计算 → MaxPool
5. GAP + FC + Argmax
6. NPU输出分类结果，CPU读取
```

**动手练习：**

```text
运行NPU单独的仿真：
cd src/npu/sim
编辑run.do，观察以下信号：
- mac_array_40x32_stream的数据流动
- pe.sv内部的乘累加过程
- conv_top.sv的状态机转换
- im2col的地址映射结果
```

#### 3. 覆盖率驱动验证

- 代码覆盖率：branch, condition, expression, statement, toggle, FSM

### 5.3 SystemVerilog Assertion（SVA）

**重点文件：** `src/dma/dma_axi_if.sv`

学习如何用SVA验证AXI协议：

```systemverilog
// 示例：AR通道握手规则
assert property (@(posedge clk)
    arvalid && !arready |=> arvalid
) else $error("ARVALID deasserted before ARREADY");

// 示例：数据通道必须在地址通道之后
assert property (@(posedge clk)
    $rose(rvalid) |-> $past(arvalid, N)
) else $error("RDATA without prior ARADDR");
```

## 第六阶段：进阶主题（第7-8周）

### 6.1 量化与训练工具链

**阅读目录：** `src/npu/export_cifar/`

**文件：**

- `train_cifar10_5x5.py` — PyTorch训练脚本
- `export_cifar10_int8_pow2_fused.py` — INT8量化 + 权重导出
- `infer_cifar10_int8_pow2_fused.py` — 量化推理验证

**关键知识点：**

- 训练后量化（Post-Training Quantization）
- INT8定点数表示：1位符号 + 7位整数（或类似方案）
- Power-of-2量化：用移位代替乘法
- 权重融合：将BN层融合到卷积权重中

### 6.2 低功耗设计

**阅读：** `docs/DESIGN_REPORT.md` 中的低功耗章节

**关键知识点：**

- 时钟门控（Clock Gating）
- 动态频率调节（DFS）
- 电源门控（Power Gating）
- 脉动阵列的天然低功耗特性：数据局部性好，减少访存

### 6.3 性能分析

理解这个NPU的性能指标：

```text
时钟频率: 100-200 MHz
MAC数量: 1280
理论峰值: 1280 × 200MHz = 256 GOPS (INT8)
实际推理: ~22,715 cycles @ 100MHz = 227 μs
每张图像: ~22,715 × 1280 = 29M MAC操作
实际性能: 29M / 227μs ≈ 128 GOPS
利用率: 128 / 256 = 50%
```

思考：为什么利用率只有50%？瓶颈在哪里？（答案：数据搬运、im2col、MaxPool等非计算阶段）

## 建议的学习顺序总结

```text
Week 1-2: AXI协议 → Crossbar → 地址映射
Week 2-3: PicoRV32 → CPU固件 → 软硬件协同
Week 3-4: DMA控制器 → SVA断言 → 验证方法
Week 4-6: NPU脉动阵列 → im2col → 完整推理流程（重点投入）
Week 6-7: SoC集成 → 测试bench → 覆盖率
Week 7-8: 量化工具链 → 性能优化 → 低功耗设计
```

这个项目的精华在于NPU的脉动阵列设计和DMA控制器的工程实现。建议把最多的时间花在理解`mac_array_40x32_stream.sv`和`dma_axi_if.sv`上——前者代表了AI芯片的核心计算架构，后者代表了高性能总线接口的设计范式。
