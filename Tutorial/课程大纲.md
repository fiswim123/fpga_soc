# CPU+NPU 异构SoC设计 — 30讲课程大纲

> 本课程以本项目为唯一教学素材，逐行讲解RTL代码，从总线协议到NPU推理，循序渐进。
> 重点模块：AXI Crossbar（4讲）、DMA控制器（6讲）、NPU子系统（8讲）。

---

## 课程总体结构

```text
模块一  总线协议与SoC互联（第1-6讲）    ← AXI Crossbar 重点
模块二  CPU子系统（第7-9讲）
模块三  DMA搬运器（第10-15讲）          ← DMA 重点
模块四  NPU神经网络加速器（第16-23讲）   ← NPU 重点
模块五  SoC集成与验证（第24-27讲）
模块六  进阶专题（第28-30讲）
```

## 模块一：总线协议与SoC互联（第1-6讲）

### 第1讲 SoC全景 — 从一张图开始

**目标：** 建立整个异构处理器的全局认知。

**教学内容：**

1. 项目背景：第九届中国研究生创芯大赛飞腾企业命题
2. SoC整体架构框图讲解
   - CPU（PicoRV32）→ AXI-Lite → Crossbar → DDR / NPU RAM / DMA CSR / NPU CSR
   - DMA作为独立主设备，直接搬运数据
   - NPU作为计算加速器，通过CSR被CPU控制
3. 地址映射表逐项讲解

   | 地址范围 | 设备 | 大小 | 用途 |
   |---------|------|------|------|
   | `0x0000_0000` | CPU ROM | 4KB | 固件代码 |
   | `0x1000_0000` | CPU RAM | 4KB | 栈/变量 |
   | `0x0000_1000 - 0x0002_0FFF` | NPU RAM | 128KB | 图像/特征图 |
   | `0x0002_1000 - 0x0002_1FFF` | DMA CSR | 4KB | DMA寄存器 |
   | `0x0003_0000 - 0x0003_0FFF` | NPU CSR | 4KB | NPU寄存器 |
   | `0x4000_0000 - 0x4003_FFFF` | DDR | 256KB | 主存 |

4. 数据流全景：一张CIFAR-10图像从DDR到分类结果的完整路径
5. 源码目录结构导航

**核心文件：** `src/soc_top.sv`（通读，建立全局印象）

**课后任务：** 手画SoC框图，标注每个接口的协议类型和位宽。

### 第2讲 AXI4协议基础 — 五通道握手

**目标：** 掌握AXI4协议的核心机制。

**教学内容：**

1. 为什么需要总线协议？CPU和外设之间的通信桥梁
2. AXI4的五个通道逐一讲解
   - AW（写地址）：`awvalid`, `awready`, `awaddr`, `awlen`, `awsize`, `awburst`
   - W（写数据）：`wvalid`, `wready`, `wdata`, `wstrb`, `wlast`
   - B（写响应）：`bvalid`, `bready`, `bresp`
   - AR（读地址）：`arvalid`, `arready`, `araddr`, `arlen`, `arsize`, `arburst`
   - R（读数据）：`rvalid`, `rready`, `rdata`, `rlast`, `rresp`
3. Valid/Ready握手规则（三条铁律）
   - 发起方拉高valid后不能撤回
   - valid和ready可以同周期完成握手
   - valid不能等待ready才拉高
4. 突发传输：INCR、FIXED、WRAP的区别
5. 4KB边界限制及其原因
6. 读写事务的完整时序图

**核心文件：** `src/dma/inc/amba_axi.svh`（信号定义）、`src/dma/inc/amba_axi_pkg.sv`（类型定义）

**课后任务：** 对照AXI规范文档，画出一次4-beat INCR读传输的时序图。

### 第3讲 AXI-Lite与协议转换桥

**目标：** 理解AXI-Lite子集和全AXI4之间的转换。

**教学内容：**

1. AXI-Lite vs AXI4的区别
   - 无burst支持（len=0, size固定）
   - 无ID标签（不支持乱序）
   - 信号更少，接口更简单
2. PicoRV32为什么用AXI-Lite？——简单CPU，简单接口
3. `axi_lite2axi.sv` 逐行讲解
   - AXI-Lite缺少的信号如何补全（id、len、size、burst）
   - 状态机：如何将单拍传输包装为burst=1的AXI4事务
4. `axi2axi_lite.sv` 反向桥讲解
   - 如何将AXI4的burst响应拆回AXI-Lite的单拍响应

**核心文件：** `src/axi_crossbar/axi_lite2axi.sv`、`src/axi_crossbar/axi2axi_lite.sv`

**课后任务：** 在仿真波形中观察CPU的一次读操作，标注信号在桥两侧的变化。

### 第4讲 AXI Crossbar（一）— 顶层架构与地址解码

**目标：** 理解4×4交叉矩阵的结构和地址路由机制。

**教学内容：**

1. 什么是Crossbar？为什么不用总线（共享仲裁）？
   - 总线：一次只能一个主设备访问 → 带宽瓶颈
   - Crossbar：多对主从可以同时通信 → 高带宽
2. `axicb_crossbar_top.sv` 顶层结构讲解
   - 4个Slave端口（连接主设备：CPU、DMA、...）
   - 4个Master端口（连接从设备：DDR、NPU RAM、DMA CSR、NPU CSR）
   - 每个Slave端口有独立的地址解码器
3. 地址解码逻辑
   - 如何根据`araddr`/`awaddr`选择目标Master端口
   - 地址范围配置参数：`ADDR_BASE`、`ADDR_SIZE`、`ADDR_MASK`
   - 未命中地址的处理（DECERR响应）
4. ID Mask机制
   - 每个Slave端口分配唯一ID前缀（CPU=0x10, DMA=0x20）
   - 响应路由时根据ID前缀判断返回给哪个Slave

**核心文件：** `src/axi_crossbar/axicb_crossbar_top.sv`

**课后任务：** 写出CPU访问`0x4000_0000`（DDR）和DMA访问`0x0000_1000`（NPU RAM）时，信号分别经过Crossbar的哪些路径。

### 第5讲 AXI Crossbar（二）— 仲裁器与乱序完成

**目标：** 深入理解仲裁策略和乱序完成机制。

**教学内容：**

1. 优先级轮询仲裁器 `axicb_round_robin_arbiter.sv`
   - 四级优先级轮询：如何公平地在多个主设备之间分配带宽
   - `axicb_round_robin_priority.sv`：优先级与轮询的结合
   - 仲裁时机：地址通道仲裁 vs 数据通道仲裁
2. 交换矩阵 `axicb_switch_top.sv`
   - 如何将选中的Slave端口连接到目标Master端口
   - 数据通路的多路选择器树
3. 乱序完成 `axicb_slv_ooo.sv`
   - 为什么需要乱序？——慢设备不应阻塞快设备的响应
   - per-ID FIFO跟踪：每个outstanding事务的来源端口
   - 完成时根据ID将数据路由回正确的Slave端口
4. Outstanding事务管理
   - 每个端口的outstanding深度配置（CPU=4, DMA=4）
   - 为什么更多outstanding = 更高带宽？——隐藏访存延迟
5. 实例讲解：CPU和DMA同时访问不同从设备时的并发行为

**核心文件：** `src/axi_crossbar/axicb_round_robin_arbiter.sv`、`src/axi_crossbar/axicb_slv_ooo.sv`、`src/axi_crossbar/axicb_switch_top.sv`

**课后任务：** 分析仿真波形中CPU和DMA同时发起传输时的仲裁行为。

### 第6讲 AXI Crossbar（三）— CSR桥与完整数据通路

**目标：** 理解CSR桥接器，串联Crossbar的完整数据通路。

**教学内容：**

1. `axi2csr.sv` — AXI4到CSR桥
   - 将AXI4读写事务转换为简单的寄存器读写接口
   - NPU CSR为什么用这个桥？——NPU不需要burst，只需要单寄存器访问
   - 时序：AXI地址握手 → 产生`csr_wr_en`/`csr_rd_en` → AXI数据响应
2. `axicb_crossbar_slice.sv` — 流水线切片
   - 为什么需要寄存器切片？——切断组合逻辑路径，提高Fmax
   - 前向切片 vs 反向切片
3. `axicb_err_slave.sv` — 错误响应从设备
   - 地址未命中时返回SLVERR/DECERR
4. 完整数据通路串联
   - CPU读NPU CSR的完整路径：PicoRV32 → AXI-Lite → Lite2AXI → Crossbar(slv0→mst3) → AXI2CSR → NPU CSR
   - DMA写NPU RAM的完整路径：DMA FSM → DMA AXI IF → Crossbar(slv1→mst1) → NPU RAM

**核心文件：** `src/axi_crossbar/axi2csr.sv`、`src/axi_crossbar/axicb_crossbar_slice.sv`、`src/axi_crossbar/axicb_err_slave.sv`

**目标：** 理解CPU如何连接到SoC总线。

**教学内容：**

   - PicoRV32核心 + 存储器路由器 + AXI-Lite适配器

4. 本地存储器实例化
   - 4KB ROM：`rom.sv`，加载`instr_data.dat`

## 模块三：DMA搬运器（第10-15讲）

### 第10讲 DMA架构总览 — 7层层次化设计

**目标：** 建立DMA控制器的全局认知。

**教学内容：**

1. 为什么需要DMA？——CPU搬数据太慢，且搬数据时不能做其他事
2. DMA在SoC中的角色
   - 独立于CPU的主设备
   - CPU只负责配置，DMA负责执行
   - 典型流程：CPU配置描述符 → DMA启动 → CPU去做别的 → DMA完成中断
3. 本项目DMA的7层模块层次

   ```text
   dma_axi_top          ← 最外层：信号打包
     dma_axi_wrapper     ← CSR + 功能逻辑集成
       dma_csr            ← AXI-Lite CSR寄存器文件
       dma_func_wrapper   ← 功能核心
         dma_fsm           ← 4状态控制器
         dma_streamer ×2   ← 读/写突发引擎
         dma_axi_if        ← AXI4 Master接口
         dma_fifo          ← 数据缓冲
   ```

4. 双描述符设计
   - DESC0和DESC1独立配置
   - 用途：链式传输、ping-pong缓冲
5. DMA CSR寄存器映射表

   | 偏移 | 寄存器 | 说明 |
   |------|--------|------|
   | `0x00` | CONTROL | go, abort, max_burst |
   | `0x08` | STATUS | done, error, signature(0xCAFE) |
   | `0x10` | ERROR_ADDR | 错误事务地址 |
   | `0x18` | ERROR_STATS | 错误类型/来源 |
   | `0x20` | SRC_ADDR_0 | 描述符0源地址 |
   | `0x30` | DST_ADDR_0 | 描述符0目的地址 |
   | `0x40` | NUM_BYTES_0 | 描述符0传输字节数 |
   | `0x50` | CFG_0 | 描述符0配置 |

**核心文件：** `src/dma/dma_axi_top.sv`、`src/dma/dma_axi_wrapper.sv`

**课后任务：** 画出DMA的模块层次图，标注每层的接口方向（主/从）。

### 第11讲 DMA CSR — 寄存器文件的实现

**目标：** 深入理解AXI-Lite CSR寄存器文件的硬件实现。

**教学内容：**

1. `dma_csr.sv` 逐段讲解
2. AXI-Lite写通道处理
   - 写地址握手：`i_awvalid` && `o_awready`
   - 写数据握手：`i_wvalid` && `o_wready`
   - 写响应生成：`o_bvalid` && `i_bready`
   - 地址解码：`i_awaddr[7:0]` → 选择写入哪个寄存器
3. AXI-Lite读通道处理
   - 读地址握手：`i_arvalid` && `o_arready`
   - 读数据输出：`o_rvalid` && `i_rready`
   - 地址解码：`i_araddr[7:0]` → 选择读取哪个寄存器
4. 寄存器读写逻辑
   - CONTROL寄存器：只写（go/abort触发脉冲）
   - STATUS寄存器：只读（由DMA FSM驱动）
   - DESC寄存器：可读可写
   - 硬件清零：go信号自动清零（单周期脉冲）
5. 两个描述符的寄存器组织
   - `o_dma_desc_src_addr_src_addr[0]` 和 `[1]` 分别对应DESC0和DESC1
   - 如何用二维packed array实现多描述符

**核心文件：** `src/dma/dma_csr.sv`

**课后任务：** 用ModelSim的`axil_dma_write`任务写DESC0的SRC_ADDR，观察`o_dma_desc_src_addr_src_addr[0]`的变化。

### 第12讲 DMA FSM — 四状态控制器

**目标：** 理解DMA的状态机设计和描述符调度。

**教学内容：**

1. `dma_fsm.sv` 逐段讲解
2. 四个状态详解
   - `IDLE`：等待go信号
   - `CFG`：检查描述符有效性，加载配置
   - `RUN`：驱动读/写streamer执行传输
   - `DONE`：设置done标志，等待CPU读取
3. 状态转换条件

   ```text
   IDLE → CFG:  go信号 && 至少一个描述符enable
   CFG  → RUN:  描述符检查通过
   RUN  → DONE: 所有描述符的读写都完成
   DONE → IDLE: CPU读取STATUS（或新的go信号）
   ```

4. 描述符调度逻辑
   - `rd_desc_done_ff` / `wr_desc_done_ff`：跟踪每个描述符的读写完成状态
   - `rd_idx_ff` / `wr_idx_ff`：当前正在处理的描述符索引
   - `pending_desc`：是否还有未处理的描述符
5. Abort处理
   - abort信号如何中断正在进行的传输
   - 清理逻辑：清除FIFO、复位streamer

**核心文件：** `src/dma/dma_fsm.sv`

**课后任务：** 仿真中force `dma_ctrl_i.go = 1`，观察FSM从IDLE到DONE的完整状态转换。

### 第13讲 DMA Streamer — 读写突发引擎

**目标：** 理解DMA如何将大块数据拆分为AXI突发传输。

**教学内容：**

1. `dma_streamer.sv` 整体结构
   - 两个实例：一个负责读（从源地址读到FIFO），一个负责写（从FIFO写到目的地址）
   - 读写可以并行工作（流水线化）
2. 突发拆分逻辑
   - 输入：总传输字节数、最大突发长度
   - 计算：需要多少次burst，每次burst多少拍
   - 4KB边界检测：如果burst跨越4KB边界，自动拆分
3. 地址递增逻辑
   - INCR模式：每拍地址递增（+1, +2, +4, +8取决于size）
   - FIXED模式：地址不变（用于FIFO类外设）
4. 字节使能处理
   - 非对齐起始地址：第一拍的wstrb需要屏蔽低位字节
   - 非对齐结束地址：最后一拍的wstrb需要屏蔽高位字节
5. 完成判断
   - `last_burst`：最后一次burst的标志
   - 流入`dma_fsm`的完成信号

**核心文件：** `src/dma/dma_streamer.sv`

**课后任务：** 仿真Test 3（burst=4）和Test 14（4KB边界），对比streamer的地址递增行为。

### 第14讲 DMA AXI接口 — 五通道引擎与SVA断言

**目标：** 这是DMA最复杂的模块，理解AXI4 Master接口的完整实现。

**教学内容：**

1. `dma_axi_if.sv` 顶层结构
   - 5个通道的状态机（AW/W/B/AR/R各自独立控制）
   - Outstanding事务追踪
2. 写地址通道（AW）
   - `awvalid`/`awready`握手
   - 突发参数设置：`awlen`、`awsize`、`awburst`
3. 写数据通道（W）
   - `wvalid`/`wready`握手
   - `wlast`信号：最后一拍的标志
   - `wstrb`字节使能
4. 写响应通道（B）
   - `bresp`检查：SLVERR/DECERR的处理
   - 错误记录逻辑
5. 读地址通道（AR）
   - 与AW通道类似的握手逻辑
6. 读数据通道（R）
   - `rlast`信号：判断burst结束
   - `rresp`检查
7. SVA断言逐条讲解
   - AXI协议合规性检查
   - 例如：valid不能在ready之前撤回
   - 例如：burst长度必须与awlen/arlen一致

4. 硬件-软件对齐

### 第29讲 低功耗设计 — 理论与实践

**目标：** 理解异构SoC的低功耗设计策略。

**教学内容：**

1. 功耗来源分析
   - 动态功耗：P = α × C × V² × f
   - 静态功耗：漏电流
2. 时钟门控（Clock Gating）
   - 原理：不需要工作时关闭时钟
   - NPU中的应用：未使用的子阵列关闭时钟
   - RTL实现：`clk_en`信号 + latch-based clock gate
3. 动态频率调节（DFS）
   - 原理：根据负载调整时钟频率
   - NPU中：推理时用高频，空闲时用低频
   - 实现：时钟分频器 + 频率切换逻辑
4. 电源门控（Power Gating）
   - 原理：完全切断不需要工作的模块的电源
   - 挑战：状态保存、唤醒延迟
5. 脉动阵列的天然低功耗特性
   - 数据局部性：每个PE只与相邻PE通信
   - 无全局数据广播：减少线网翻转
   - 权重驻留：减少权重读取的功耗
6. 本项目中的低功耗设计（参考`docs/DESIGN_REPORT.md`）

**核心文件：** `docs/DESIGN_REPORT.md`（低功耗章节）

**课后任务：** 分析NPU在Conv1 vs Conv2阶段的功耗差异（通过toggle coverage估算）。

### 第30讲 性能分析与优化 — 从瓶颈到加速

**目标：** 学会分析异构SoC的性能瓶颈并提出优化方案。

**教学内容：**

1. 性能指标计算

   ```text
   时钟频率: 100-200 MHz
   MAC数量: 1280
   理论峰值: 1280 × 200MHz = 256 GOPS (INT8)
   实际推理: ~22,715 cycles @ 100MHz = 227 μs
   每张图像: ~22,715 × 1280 ≈ 29M MAC操作
   实际性能: 29M / 227μs ≈ 128 GOPS
   利用率: 128 / 256 = 50%
   ```

2. 瓶颈分析
   - 计算利用率只有50%，另一半时间在做什么？
   - 数据搬运开销：DMA传输、im2col变换
   - 非计算层：MaxPool、GAP不使用脉动阵列
   - 层间气泡：Conv1→Conv2的切换延迟
3. 优化方向
   - 增加脉动阵列规模：更多PE → 更高并行度
   - 流水线化层间处理：Conv2的im2col与Conv1的MaxPool重叠
   - 增加DMA带宽：更宽的数据总线、更多outstanding
   - 片上存储优化：更大的SA RAM减少分块次数
4. 与工业界NPU的对比
   - Google TPU v1：92 TOPS (INT8)，脉动阵列256×256
   - 华为Ascend：达芬奇架构，3D Cube计算单元
   - 本项目的设计空间探索
5. 总结与展望
   - 本项目学到了什么？
   - 如何在此基础上扩展？
   - 下一步学习方向：更复杂的网络、更大的阵列、FPGA实现

**核心文件：** `docs/DESIGN_REPORT.md`（性能分析章节）、`docs/NPU_DESIGN_REPORT.md`（时序分析）

**课后任务：** 提出至少3个具体的优化方案，估算每个方案的性能提升。
