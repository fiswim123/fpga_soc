# 第二十九讲：低功耗设计 — 理论与实践

---

## 课程目标

本讲从CMOS电路功耗的基本公式出发，系统讲解FPGA SoC中三种核心低功耗技术：时钟门控（Clock Gating）、动态频率调整（DFS）和电源门控（Power Gating），并分析脉动阵列架构天然的低功耗优势。

**学完本讲你将掌握：**

1. 动态功耗与静态功耗的来源和计算公式
2. 时钟门控（ICG）的原理、RTL实现和分层策略
3. 动态频率调整（DFS）的无毛刺切换设计
4. 电源门控（PG）的隔离、保持和唤醒流程
5. 脉动阵列的局部通信特性如何天然降低功耗
6. 三级低功耗体系的协同工作机制

## 1. CMOS功耗基础

### 1.1 动态功耗

CMOS电路的动态功耗来源于信号翻转时对负载电容的充放电：

```
P_dynamic = alpha × C × V_DD^2 × f

其中:
  alpha = 活动因子 (0~1, 信号翻转概率)
  C     = 等效负载电容 (fF)
  V_DD  = 供电电压 (V)
  f     = 时钟频率 (Hz)
```

**各因子的影响：**

```
┌─────────────────────────────────────────────────────┐
│                 动态功耗组成                          │
│                                                     │
│  P = alpha × C × V^2 × f                           │
│       │      │     │      │                         │
│       │      │     │      └── 频率: 200MHz→100MHz    │
│       │      │     │          功耗减半               │
│       │      │     │                                 │
│       │      │     └── 电压: 1.0V→0.8V              │
│       │      │          功耗降为 0.64× (平方关系!)   │
│       │      │                                       │
│       │      └── 电容: 由工艺和设计决定              │
│       │           翻转的门越多, C越大                │
│       │                                             │
│       └── 活动因子: 信号翻转频率                     │
│            时钟门控将alpha降为0!                      │
└─────────────────────────────────────────────────────┘
```

### 1.2 静态功耗（漏电功耗）

即使电路不翻转，CMOS晶体管也存在亚阈值漏电流：

```
P_static = V_DD × I_leak

I_leak 主要来源:
  1. 亚阈值漏电 (Subthreshold): Vgs < Vth 时的弱反型电流
  2. 栅极漏电 (Gate Leakage): 栅氧化层隧穿电流
  3. 结漏电 (Junction Leakage): PN结反向偏置电流
```

**工艺缩放的影响：**

```
工艺节点    典型V_DD    相对漏电
  180nm      1.8V        1×
   65nm      1.2V       10×
   28nm      0.9V       50× (本设计目标工艺)
   7nm       0.75V     200×+

结论: 工艺越先进, 漏电占比越大, 电源门控越重要!
```

### 1.3 总功耗

```
P_total = P_dynamic + P_static

         ┌────────────────────────────────┐
         │  高频/大电压场景 (高性能模式)    │
         │  P_dynamic 占比 70-80%          │
         │  P_static  占比 20-30%          │
         ├────────────────────────────────┤
         │  低频/低电压场景 (低功耗模式)    │
         │  P_dynamic 占比 40-50%          │
         │  P_static  占比 50-60%          │
         ├────────────────────────────────┤
         │  深度睡眠场景 (时钟门控+电源门控) │
         │  P_dynamic ≈ 0                  │
         │  P_static  占比 100%            │
         └────────────────────────────────┘
```

### 设计视角：为什么这样设计？

低功耗设计的核心是用最小的面积和性能代价换取最大的功耗节省。本节分析三级低功耗体系背后的设计动机。

**核心问题 1：为什么需要三级功耗管理？**

```
  单一技术的局限性:

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  仅用时钟门控 (CG):                                     │
  │  ├── P_dynamic = 0 (alpha = 0)  ✓                      │
  │  ├── P_static 仍然存在  ✗                              │
  │  ├── 28nm 工艺漏电占比 ~30-50%                         │
  │  └── 空闲时功耗 = 0 + 0.4 = 0.4 (基准)                │
  │                                                         │
  │  仅用 DFS:                                              │
  │  ├── P_dynamic ∝ f (线性降低)  ✓                       │
  │  ├── 但不能降到 0 (需要维持时钟)  ✗                     │
  │  ├── P_static 仍然存在  ✗                              │
  │  └── 最低功耗 ≈ 0.125 + 0.4 = 0.525 (基准)            │
  │                                                         │
  │  仅用电源门控 (PG):                                     │
  │  ├── P_dynamic + P_static ≈ 0  ✓                       │
  │  ├── 但唤醒延迟长 (~32 cycles)  ✗                      │
  │  ├── 频繁开关功耗 (rush current)  ✗                    │
  │  └── 不适用于短时间空闲                                  │
  │                                                         │
  │  三级组合:                                              │
  │  ├── CG: 短时间空闲 (1 cycle 响应)                     │
  │  ├── DFS: 中等负载变化 (几 cycles 响应)                │
  │  ├── PG: 长时间空闲 (~32 cycles 响应)                  │
  │  └── 总功耗 = 0.135 (基准) → 节省 86.5%               │
  │                                                         │
  └─────────────────────────────────────────────────────────┘

  三级体系的协同逻辑:

  空闲时间        响应速度        选择技术        功耗节省
  ────────────────────────────────────────────────────────
  1-10 cycles     即时            时钟门控        ~60-77%
  10-1000 cycles  几 cycles       DFS             ~50-75%
  > 1000 cycles   ~32 cycles      电源门控        ~95%
  ────────────────────────────────────────────────────────

  选择依据: 空闲时间 > 唤醒延迟 × 10 → 使用更深的省电级别
```

**核心问题 2：为什么时钟门控排在第一位？**

```
  时钟门控的优先级分析:

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  原因 1: 响应速度最快                                   │
  │  ├── CG 响应: 1 时钟周期 (即时生效)                     │
  │  ├── DFS 响应: 几个时钟周期 (无毛刺切换)                │
  │  ├── PG 响应: ~32 时钟周期 (电源稳定)                   │
  │  └── CG 可以在每个周期动态开关, 无需预判                │
  │                                                         │
  │  原因 2: 面积代价最小                                   │
  │  ├── CG: 每个时钟域 1 个 Latch + 1 个 AND 门           │
  │  ├── DFS: 分频器 + 多路选择器 + 同步器                  │
  │  ├── PG: 休眠晶体管 + 隔离单元 + 保持寄存器 + 控制 FSM  │
  │  └── CG 的面积开销 ≈ 0.1% (可忽略)                     │
  │                                                         │
  │  原因 3: 设计侵入性最低                                 │
  │  ├── CG: 只需在时钟树上插入 ICG 单元                    │
  │  ├── DFS: 需要修改时钟树拓扑                            │
  │  ├── PG: 需要电源域划分 + 隔离策略 + 保持寄存器         │
  │  └── CG 对现有设计的修改最小                            │
  │                                                         │
  │  原因 4: 覆盖面最广                                     │
  │  ├── CG: 适用于任何有时钟的模块                         │
  │  ├── DFS: 仅适用于可降频的模块                          │
  │  ├── PG: 仅适用于可断电的模块                           │
  │  └── CG 是最通用的低功耗技术                            │
  │                                                         │
  └─────────────────────────────────────────────────────────┘

  设计优先级:
  1. 先实现时钟门控 (投入产出比最高)
  2. 再实现 DFS (进一步降低活跃功耗)
  3. 最后实现电源门控 (消除漏电, 但复杂度最高)
```

**核心问题 3：为什么选择 Latch-based ICG 而不是直接 AND？**

```
  直接 AND 的问题:

  clk:   ──┐  ┌──┐  ┌──┐  ┌──
            └──┘  └──┘  └──┘  └──
  en:    ────────────┐
                     └────────────  (en 在 clk 高电平变化)
  直接:  ──┐  ┌──┐  ┌┘  ┌──┐  ┌──
            └──┘  └─┘   └──┘  └──
                     ↑
                     毛刺! (glitch)

  Latch-based ICG 的解决方案:

  clk:      ──┐  ┌──┐  ┌──┐  ┌──
               └──┘  └──┘  └──┘  └──
  en:       ────────────┐
                        └────────────
  latch:    ──────────────┐          (在 clk 低电平锁存 en)
                          └──────────
  gated:    ──┐  ┌──┐  ┌──┐  ┌──     (干净的门控时钟)
               └──┘  └──┘  └──┘  └──

  原理:
  · Latch 在 clk 低电平期间透明, 采样 en 值
  · clk 高电平期间 latch 锁存, en 变化不影响输出
  · gated_clk = clk & en_latch → 输出时钟无毛刺
```

### 设计视角：如何从零开始设计？

设计一个 FPGA SoC 的低功耗系统需要从功耗预算出发, 逐步实现各级功耗管理。以下是五步设计流程。

**步骤 1：建立功耗模型**

```
  功耗分析流程:

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  1. 识别功耗源                                          │
  │     · 动态功耗: P = alpha × C × V² × f                 │
  │     · 静态功耗: P = V × I_leak                          │
  │     · 开关功耗: 短路电流、rush current                   │
  │                                                         │
  │  2. 估算各模块功耗占比                                  │
  │     · NPU 脉动阵列: ~60% (大量 MAC 单元)                │
  │     · NPU 存储器: ~20% (SRAM 读写)                      │
  │     · DMA 控制器: ~10% (数据搬运)                       │
  │     · CPU + Crossbar: ~10% (控制逻辑)                   │
  │                                                         │
  │  3. 确定功耗目标                                        │
  │     · 全速推理: <= 0.5W                                 │
  │     · 空闲: <= 0.05W                                    │
  │     · 深度睡眠: <= 0.01W                                │
  │                                                         │
  └─────────────────────────────────────────────────────────┘
```

**步骤 2：实现时钟门控（Level 1）**

```
  时钟门控实现步骤:

  Step 1: 设计 ICG 单元
  · Latch (negedge clk) + AND 门
  · 确保输出无毛刺

  Step 2: 在模块边界插入 ICG
  · 每个 NPU Core 一个独立 ICG
  · 控制信号来自 CSR 寄存器 (NPU_CORE_MASK)

  Step 3: 生成门控使能信号
  · core_clk_en[i] = npu_active & core_mask[i]
  · npu_active: NPU 有推理任务时为 1
  · core_mask[i]: CSR 可配置, 动态选择活跃 Core

  Step 4: 验证门控行为
  · 仿真: 检查 gated_clk 在 en=0 时恒为低
  · 覆盖率: 确保 en=0 和 en=1 两种状态都被覆盖
  · 时序: 确保 ICG 不引入时序违规
```

**步骤 3：实现动态频率调整（Level 2）**

```
  DFS 实现步骤:

  Step 1: 设计无毛切分频器
  · 支持 /1, /2, /4, /8 四种分频比
  · 在时钟低电平采样分频比变化
  · 使用两级同步器消除亚稳态
  · 旧分频器完成当前周期后才切换

  Step 2: 通过 CSR 控制分频比
  · NPU_CLK_DIV 寄存器: [1:0] 分频比选择
  · CPU 通过 AXI-Lite 写入

  Step 3: 选择合适的分频模式
  · 高性能 (/1): 大型 CNN 推理
  · 均衡 (/2): 中等网络
  · 低功耗 (/4): 小型网络
  · 待机 (/8): 无推理任务

  Step 4: 验证无毛切切换
  · 仿真: 在切换过程中检查输出时钟无毛刺
  · 功能: 切换后 NPU 推理结果正确
```

**步骤 4：实现电源门控（Level 3）**

```
  电源门控实现步骤:

  Step 1: 划分电源域
  · 每个 NPU Core 一个独立电源域
  · 电源域边界需要隔离单元

  Step 2: 设计隔离单元
  · 断电时将输出钳位到已知值 (0 或 1)
  · 防止浮空输出引起下游逻辑误翻转

  Step 3: 设计保持寄存器
  · 关键状态 (FSM 状态、配置寄存器) 需要保持
  · 使用低漏电备份锁存器

  Step 4: 设计电源门控控制器
  · 断电流程: 完成计算 → 排空流水线 → 保存状态 → 隔离 → 断电
  · 上电流程: 释放隔离 → 等待稳定 → 恢复状态 → 就绪
  · 延迟: ~32 cycles (电源稳定)

  Step 5: 通过 CSR 控制
  · NPU_PG_EN 寄存器: 每 bit 控制一个 Core 的电源

  注意: FPGA 上电源门控有限制
  · FPGA 电源域固定, 无法像 ASIC 那样细粒度控制
  · 可用部分重配置 (Partial Reconfiguration) 替代
  · 本设计主要面向 ASIC, FPGA 上仅验证控制逻辑
```

**步骤 5：系统级功耗管理**

```
  场景化功耗管理策略:

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  策略 1: 任务感知调度                                   │
  │  · CPU 检测推理任务的网络规模                           │
  │  · 根据网络规模自动选择功耗模式                         │
  │  · 大型 CNN → 高性能模式 (8核@200MHz)                   │
  │  · 小型 CNN → 低功耗模式 (2核@50MHz)                    │
  │                                                         │
  │  策略 2: 空闲检测                                       │
  │  · NPU 完成推理后自动进入空闲状态                       │
  │  · 空闲超过阈值 → 降频 (DFS)                            │
  │  · 空闲超过更长阈值 → 断电 (PG)                         │
  │                                                         │
  │  策略 3: 预测性唤醒                                     │
  │  · CPU 提前发送唤醒信号 (在需要 NPU 之前)               │
  │  · NPU 在电源稳定期间完成状态恢复                       │
  │  · 计算开始时 NPU 已就绪, 无额外延迟                    │
  │                                                         │
  └─────────────────────────────────────────────────────────┘
```

### 设计视角：架构模式与原则

低功耗设计中有多种可复用的模式。掌握这些可以系统化地降低 SoC 功耗。

**模式 1：层次化功耗管理模式（Hierarchical Power Management Pattern）**

```
  核心思想: 从系统级到核级逐层细化功耗控制

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  层次结构:                                              │
  │                                                         │
  │  System Level (SoC)                                     │
  │  ├── 全局时钟门控 (整个 NPU 子系统)                     │
  │  ├── 全局 DFS (整个 NPU 子系统)                         │
  │  └── 全局 PG (整个 NPU 子系统, 极端情况)                │
  │      │                                                  │
  │      ├── Core Level (每个 NPU Core)                     │
  │      │   ├── 独立时钟门控 (Core Mask)                   │
  │      │   ├── 独立电源门控 (Core PG_EN)                  │
  │      │   └── 无独立 DFS (所有 Core 共享频率)            │
  │      │      │                                           │
  │      │      ├── PE Level (每个 PE)                      │
  │      │      │   ├── 天然低活动因子 (数据驱动)           │
  │      │      │   └── 无独立门控 (开销太大)               │
  │      │      │                                           │
  │      │      └── 存储器 (SRAM)                           │
  │      │          ├── 睡眠模式 (低漏电)                   │
  │      │          └── 输出使能 (减少翻转)                 │
  │                                                         │
  │  设计原则:                                              │
  │  · 每层的控制粒度不同 (越底层越细, 但开销越大)          │
  │  · 只在面积开销可接受的层级插入门控                     │
  │  · 控制信号从上层向下传递 (CSR → Core → PE)            │
  │                                                         │
  └─────────────────────────────────────────────────────────┘
```

**模式 2：无毛切时钟切换模式（Glitch-Free Clock Switching Pattern）**

```
  核心思想: 在切换时钟源或分频比时, 确保输出时钟无毛刺

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  无毛切切换的四要素:                                    │
  │                                                         │
  │  1. 在安全窗口采样                                      │
  │     · 在时钟低电平期间采样新的分频比/时钟源             │
  │     · 避免在时钟高电平期间切换 (可能产生窄脉冲)         │
  │                                                         │
  │  2. 同步器消除亚稳态                                    │
  │     · 使用两级 D 触发器同步异步信号                     │
  │     · 亚稳态恢复时间 = 2 个时钟周期                    │
  │                                                         │
  │  3. 等待旧周期完成                                      │
  │     · 旧分频器完成当前分频周期后才切换                  │
  │     · 避免在分频周期中间切换 (可能产生不完整周期)       │
  │                                                         │
  │  4. 互斥使能                                            │
  │     · 旧时钟禁用和新时钟使能之间有保护间隔              │
  │     · 避免两个时钟同时驱动 (竞争冒险)                   │
  │                                                         │
  │  时序示意:                                              │
  │  clk_in:  ──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──           │
  │              └──┘  └──┘  └──┘  └──┘  └──┘  └──         │
  │  div_sel: ──────[/1 → /2]──────────────────            │
  │  采样:          ↑ (低电平采样新分频比)                   │
  │  同步:              ↑↑ (2 周期同步)                     │
  │  旧周期完成:              ↑ (当前 /1 周期结束)          │
  │  切换:                      ↑ (干净切换到 /2)           │
  │  clk_out: ──┐  ┌────┐  ┌────┐  ┌────                  │
  │              └──┘    └──┘    └──┘    └──               │
  │              (/1)        (/2, 干净切换)                  │
  │                                                         │
  └─────────────────────────────────────────────────────────┘
```

**模式 3：功耗-性能权衡模式（Power-Performance Trade-off Pattern）**

```
  核心思想: 根据工作负载动态调整功耗与性能的平衡点

  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  功耗-性能曲线:                                         │
  │                                                         │
  │  性能 ▲                                                 │
```

## 2. 时钟门控（Clock Gating）

### 2.1 基本原理

时钟门控是最基础、最广泛使用的低功耗技术。核心思想：当模块空闲时，切断其时钟信号，消除所有寄存器的无效翻转。

```
未门控:
  clk ──────┬──────────────────► FF[0]
            ├──────────────────► FF[1]
            ├──────────────────► FF[2]
            └──────────────────► FF[N]
  即使数据不变, 时钟每周期仍驱动所有FF翻转 (无效功耗)

门控后:
  clk ──┐ ICG ─┬──────────────► FF[0]
        │      ├──────────────► FF[1]
  en ───┘      ├──────────────► FF[2]
               └──────────────► FF[N]
  en=0时, gated_clk恒低, 所有FF停止翻转 (零动态功耗)
```

### 2.2 ICG（集成时钟门控单元）设计

**源码位置：** `docs/DESIGN_REPORT.md` 第3.6.1节

```verilog
// ICG (Integrated Clock Gating) cell - 避免毛刺
always @(negedge clk or negedge rst_n) begin
    if (!rst_n)
        clk_en_latch <= 1'b0;
    else
        clk_en_latch <= clk_en;  // 仅在时钟低电平锁存使能
end

assign gated_clk = clk & clk_en_latch;  // AND门产生门控时钟
```

**关键设计要点：**

```
          ┌─────────────────────────────────────────────┐
          │            ICG 单元内部                       │
          │                                             │
  clk ────┤──┐                                         │
          │  │  ┌─────────┐                             │
          │  └─►│ Latch   │──┐                          │
          │     │ (低电平  │  │  ┌─────┐                │
  en ─────┼────►│  锁存)  │  └─►| AND │──► gated_clk   │
          │     └─────────┘     └─────┘                │
          │         ▲              ▲                    │
          │         │              │                    │
          │    仅在clk=0时    gated_clk = clk & en_latch│
          │    更新使能       (en=0时恒为0, 无翻转)      │
          └─────────────────────────────────────────────┘

为什么用Latch而不是直接AND?
  直接: gated_clk = clk & en
  问题: en在clk高电平变化时产生毛刺(glitch)
  Latch: en在clk低电平被锁存, 高电平期间稳定
         → 输出时钟干净无毛刺
```

### 2.3 分层时钟门控架构

本设计采用三级时钟门控，从系统级到核级逐层细化：

```
                    clk_sys (200MHz)
                      │
              ┌───────┴───────┐
              │   ICG_NPU     │  ← NPU顶层门控
              │   en: NPU活跃  │     (推理任务使能)
              └───────┬───────┘
                      │ clk_npu_top
          ┌───────────┼───────────┐
          │           │           │
    ┌─────┴─────┐ ┌──┴──┐ ┌─────┴─────┐
    │ ICG_CORE0 │ │ ... │ │ ICG_CORE7 │  ← 单核门控
    │ en: mask[0]│ │     │ │ en: mask[7]│    (Core Mask)
    └─────┬─────┘ └──┬──┘ └─────┬─────┘
          │           │           │
      clk_core0   clk_core... clk_core7
          │           │           │
    ┌─────┴─────┐     │     ┌─────┴─────┐
    │ 脉动阵列  │     │     │ 脉动阵列  │
    │ PE[0:255] │     │     │ PE[0:255] │
    └───────────┘     │     └───────────┘
```

**CSR控制寄存器：**

```verilog
// NPU_CLK_DIV: [1:0] 分频比控制
// NPU_CORE_MASK: [7:0] 每bit使能一个Core
// NPU_PG_EN: [7:0] 每bit控制一个Core的电源门控

// 门控使能生成
assign core_clk_en[i] = npu_active & core_mask[i];
```

### 2.4 功耗节省分析

```
假设NPU有8个Core, 当前任务仅需4个Core:

未门控:
  8 Core × 256 PE × 每PE ~20个FF = 40960 个FF
  全部时钟翻转, alpha = 1.0

门控后:
  4 Core活跃: 20480 FF, alpha = 1.0 (正常工作)
  4 Core门控: 20480 FF, alpha = 0.0 (零翻转)
  总alpha ≈ 0.5 → 动态功节省 ~50%

实测数据 (docs/DESIGN_REPORT.md 第4.4.1节):
┌────────────────┬───────────┬────────────┐
│ 工作模式        │ 总翻转率  │ 归一化功耗  │
├────────────────┼───────────┼────────────┤
│ 推理全速 (8核)  │ 100%      │ 1.00       │
│ 推理半速 (4核)  │ 68.3%     │ 0.68       │
│ 推理轻载 (2核)  │ 45.1%     │ 0.45       │
│ 空闲 (0核门控)  │ 22.7%     │ 0.23       │
│ 深度睡眠(全门控)│ 5.2%      │ 0.05       │
└────────────────┴───────────┴────────────┘
```

## 3. 动态频率调整（Dynamic Frequency Scaling, DFS）

### 3.1 原理

根据计算负载动态调整工作频率。高负载时提高频率满足实时性，低负载时降低频率节省功耗。

```
P_dynamic ∝ f

频率减半 → 动态功耗减半（电压不变时）
```

### 3.2 无毛刺分频器设计

**源码位置：** `docs/DESIGN_REPORT.md` 第3.6.2节

```verilog
module clk_divider_glitchfree (
    input  wire       clk_in,
    input  wire [1:0] div_sel,   // 00:/1, 01:/2, 10:/4, 11:/8
    output wire       clk_out
);
    // 两级同步 + 无毛刺切换
    // ...
endmodule
```

**无毛刺切换的关键：**

```
问题: 切换分频比时, 如果在时钟高电平切换,
      可能产生亚稳态或窄脉冲(glitch)

解决方案:

  ┌────────────────────────────────────────────┐
  │  1. 在时钟低电平期间采样新的分频比           │
  │  2. 使用两级同步器消除亚稳态                 │
  │  3. 旧分频器完成当前周期后才切换              │
  │  4. 切换完成后置位"切换完成"标志              │
  └────────────────────────────────────────────┘

时序示意:
  clk_in:  ──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──
             └──┘  └──┘  └──┘  └──┘  └──┘  └──

  div_sel:  ──────[从/1切换到/2]───────────────

  clk_out:  ──┐  ┌────┐  ┌────┐  ┌────┐  ┌──
              └──┘    └──┘    └──┘    └──┘  └──
              (/1模式)    (/2模式, 干净切换)
```

### 3.3 DFS工作模式

**源码位置：** `docs/DESIGN_REPORT.md` 第1.6.2节

| 工作模式 | 分频比 | NPU频率 | 相对功耗 | 适用场景 |
|----------|--------|---------|----------|----------|
| 高性能 | /1 | 200 MHz | 100% | ResNet/大型CNN |
| 均衡 | /2 | 100 MHz | 50% | 中等规模网络 |
| 低功耗 | /4 | 50 MHz | 25% | LeNet/小型网络 |
| 待机 | /8 | 25 MHz | 12.5% | 无推理任务 |

**CSR配置方式：**

```c
// CPU通过AXI-Lite写NPU_CLK_DIV寄存器
// 地址: NPU_REG_BASE + 0x2C

// 设置为高性能模式 (200MHz)
npu_write_reg(NPU_CLK_DIV, 0x00);  // div_sel = 00 → /1

// 设置为均衡模式 (100MHz)
npu_write_reg(NPU_CLK_DIV, 0x01);  // div_sel = 01 → /2

// 设置为低功耗模式 (50MHz)
npu_write_reg(NPU_CLK_DIV, 0x02);  // div_sel = 10 → /4
```

### 3.4 DFS功耗实测

**源码位置：** `docs/DESIGN_REPORT.md` 第4.4.2节

以ResNet-18推理为负载：

```
NPU频率    归一化功耗    推理时间     能耗(功耗×时间)    能效比
200 MHz     1.00         4.11 ms      1.00              1.00
100 MHz     0.51         8.20 ms      1.02              0.98
 50 MHz     0.26        16.38 ms      1.04              0.96

观察:
  - 功耗近乎线性下降 (略优于线性, 因SRAM等固定开销)
  - 能耗基本恒定 (功耗×时间 ≈ 常数)
  - 低频时能效比略降 (~4%), 因漏电占比增大

结论:
  DFS主要节省"瞬时功耗"而非"总能耗"
  适用于散热受限或需要降低峰值功耗的场景
```

## 4. 电源门控（Power Gating）

### 4.1 原理

在空闲模块的电源路径上串联休眠晶体管（Sleep Transistor），彻底切断供电：

```
           V_DD
            │
     ┌──────┴──────┐
     │ Header      │  ← 高阈值PMOS休眠晶体管
     │ Sleep FET   │     (High-Vt, 低漏电)
     │ (PG_EN=0时  │
     │  关断)      │
     └──────┬──────┘
            │
     ┌──────┴──────┐
     │  逻辑模块    │  ← 被门控的NPU Core
     │  (空闲时     │     (所有逻辑断电)
     │   零功耗)    │
     └──────┬──────┘
            │
           V_SS
```

**功耗节省：**

```
时钟门控: P_dynamic = 0, 但 P_static 仍存在
电源门控: P_dynamic = 0, 且 P_static ≈ 0 (仅Sleep FET漏电)

对于28nm工艺, 电源门控可节省 ~95% 的Core级功耗
```

### 4.2 电源门控控制流程

**源码位置：** `docs/DESIGN_REPORT.md` 第3.6.3节

```
断电流程:                          上电流程:
  1. 完成当前计算                    1. CPU写NPU_PG_EN[core] = 1
  2. 排空流水线                      2. 释放隔离信号(Isolation)
  3. 保存关键状态到保持寄存器          3. 等待电源稳定 (~32 cycles)
  4. 使能隔离信号(Isolation)          4. 释放复位
  5. CPU写NPU_PG_EN[core] = 0        5. 恢复状态
  6. 切断电源开关(Header Sleep)       6. Core进入IDLE状态
```

### 4.3 隔离单元（Isolation Cell）

断电后模块输出端处于不确定态（浮空），可能引起下游逻辑的意外翻转。隔离单元在断电时将输出钳位到已知值：

```
正常工作:   output = core_output (直通)
断电隔离:   output = isolation_value (钳位到0或1)

┌──────────────────────────────────────┐
│  V_DD_off (被门控电源域)              │
│     │                                │
│  ┌──┴──┐                             │
│  │Core │──► out ──►┌──────────┐      │
│  │(断电)│          │ Isolation │──► safe_out
│  └─────┘          │ (clamp=0) │      │
│                    └──────────┘      │
│                                      │
│  PG_EN ──► 控制隔离: 断电时钳位输出   │
└──────────────────────────────────────┘
```

### 4.4 状态保持寄存器（Retention Register）

某些关键状态（如FSM状态、配置寄存器）需要在断电期间保持：

```
┌─────────────────────────────────────────┐
│  Retention Register                      │
│                                         │
│  ┌─────────┐    ┌─────────┐            │
│  │ 主锁存器  │    │ 备份锁存 │            │
│  │ (断电丢失)│    │ (低漏电  │            │
│  │          │    │  保持)   │            │
│  └────┬─────┘    └────┬────┘            │
│       │               │                 │
│       └─────┬─────────┘                 │
│             │                           │
│  save: 主→备份 (断电前)                  │
│  restore: 备份→主 (上电后)               │
└─────────────────────────────────────────┘
```

## 5. 脉动阵列的天然低功耗特性

### 5.1 局部通信 vs 全局总线

```
全局总线架构 (传统):
  ┌─────────────────────────────────────────┐
  │                                         │
  │  PE ──wire── PE ──wire── PE ──wire── PE │
  │  │           │           │           │  │
  │  └───────────┴───────────┴───────────┘  │
  │              全局数据总线                 │
  │  (所有PE共享, 长线电容大, alpha高)        │
  └─────────────────────────────────────────┘
  总线长度 ∝ N (PE数量)
  总线电容 ∝ N
  动态功耗 ∝ N × f × C_long

脉动阵列架构 (本设计):
  ┌─────────────────────────────────────────┐
  │  PE ──wire── PE ──wire── PE ──wire── PE │
  │  (仅相邻PE间短连线)                      │
  │                                         │
  │  数据逐拍"脉动"流动:                     │
  │  t=0: data在PE0                         │
  │  t=1: data→PE1, PE0处理下一个数据        │
  │  t=2: data→PE2, PE1处理...              │
  └─────────────────────────────────────────┘
  连线长度 ∝ 1 (仅相邻)
  连线电容 ∝ 1 (短线)
  动态功耗 ∝ N × f × C_short (C_short << C_long)
```

### 5.2 数据复用减少存储访问

```
传统架构 (数据需要反复读写全局存储):
  cycle 1: 从SRAM读A[0], 从SRAM读B[0] → 乘加 → 写回SRAM
  cycle 2: 从SRAM读A[0], 从SRAM读B[1] → 乘加 → 写回SRAM
  ...
  每次MAC都需要SRAM访问 → 高功耗

脉动阵列 (数据在PE间流动复用):
  cycle 1: A[0]从左端流入PE0, PE0计算
  cycle 2: A[0]→PE1, A[1]→PE0, 所有PE同时计算
  cycle 3: A[0]→PE2, A[1]→PE1, A[2]→PE0
  ...
  每个数据被多个PE复用 → SRAM访问次数大幅减少
```

**量化分析（16x16阵列，权重静止数据流）：**

```
假设: 16×16矩阵乘法, 256个PE

传统逐元素计算:
  SRAM读取次数 = 16×16×16 = 4096 次 (每次MAC读A和B)
  SRAM功耗 = 4096 × E_sram_access

脉动阵列:
  SRAM读取次数 = 16 + 16 = 32 次 (A从左端16拍流入, B从顶部16拍流入)
  SRAM功耗 = 32 × E_sram_access

SRAM访问功耗节省: 4096/32 = 128×!
```

### 5.3 权重驻留减少权重加载功耗

```
Weight Stationary 数据流:

1. 初始化阶段: 权重从SRAM加载到PE内部寄存器 (一次性)
   256 PE × 1 Byte = 256 Bytes 加载

2. 计算阶段: 权重保持不变, 仅输入数据流动
   对于多行输出(tiling), 同一组权重被复用多次

3. 权重复用次数 = TILE_ROWS = 40 (本设计)
   等效权重SRAM访问减少40×
```

### 5.4 脉动阵列功耗对比

```
┌─────────────────────────────────────────────────────┐
│  对比: 通用GPU vs 脉动阵列NPU                        │
│                                                     │
│  GPU (大量SIMD + 全局共享存储):                       │
│  · 全局数据总线: 长线, 高电容                         │
│  · 共享SRAM频繁访问: 每次MAC都需读写                  │
│  · 控制逻辑复杂: 指令译码/调度/分支预测               │
│  · 功耗: ~200W (数据中心GPU)                         │
│  · 能效: ~10 TOPS/W                                 │
│                                                     │
│  脉动阵列NPU (本设计):                               │
│  · 局部短连线: 低电容                                │
│  · 数据在PE间流动复用: SRAM访问少                    │
│  · 控制简单: 固定数据流模式                          │
│  · 功耗: ~0.5W (估算@28nm)                          │
│  · 能效: ~1.6 TOPS/W                                │
│                                                     │
│  能效比提升: ~6× (相同工艺下)                        │
└─────────────────────────────────────────────────────┘
```

## 6. 三级低功耗体系协同

### 6.1 体系架构

本设计构建了从系统级到核级的三级低功耗管理体系：

```
┌─────────────────────────────────────────────────────────┐
│                    三级低功耗体系                         │
│                                                         │
│  Level 1: 时钟门控 (Clock Gating)                       │
│  ├── 粒度: 单个NPU Core (8个独立ICG)                    │
│  ├── 控制: NPU_CORE_MASK CSR寄存器                      │
│  ├── 效果: P_dynamic = 0 (该Core)                       │
│  ├── 延迟: 1周期 (即时生效)                              │
│  └── 适用: Core级空闲 (如浅层网络仅用部分Core)           │
│                                                         │
│  Level 2: 动态频率调整 (DFS)                             │
│  ├── 粒度: 整个NPU子系统                                │
│  ├── 控制: NPU_CLK_DIV CSR寄存器                        │
│  ├── 效果: P_dynamic ∝ f (线性降低)                     │
│  ├── 延迟: 几个周期 (无毛刺切换)                         │
│  └── 适用: 负载变化 (轻推理/重推理切换)                  │
│                                                         │
│  Level 3: 电源门控 (Power Gating)                       │
│  ├── 粒度: 单个NPU Core                                 │
│  ├── 控制: NPU_PG_EN CSR寄存器                          │
│  ├── 效果: P_dynamic + P_static ≈ 0                     │
│  ├── 延迟: ~32周期 (电源稳定+状态恢复)                   │
│  └── 适用: Core长期空闲 (如推理任务间歇期)               │
└─────────────────────────────────────────────────────────┘
```

### 6.2 场景化功耗管理策略

```
场景1: 大型CNN推理 (ResNet-18)
  ┌─────────────────────────────────────┐
  │ 时钟门控: 全部8核使能 (mask=0xFF)    │
  │ DFS: 高性能模式 (/1, 200MHz)        │
  │ 电源门控: 全部上电                   │
  │ 预期功耗: 100% (基准)               │
  └─────────────────────────────────────┘

场景2: 中型CNN推理 (MobileNet)
  ┌─────────────────────────────────────┐
  │ 时钟门控: 4核使能 (mask=0x0F)       │
  │ DFS: 均衡模式 (/2, 100MHz)          │
  │ 电源门控: 4核断电 (PG_EN=0xF0)      │
  │ 预期功耗: ~34%                      │
  └─────────────────────────────────────┘

场景3: 轻量CNN推理 (LeNet-5)
  ┌─────────────────────────────────────┐
  │ 时钟门控: 2核使能 (mask=0x03)       │
  │ DFS: 低功耗模式 (/4, 50MHz)         │
  │ 电源门控: 6核断电 (PG_EN=0xFC)      │
  │ 预期功耗: ~11%                      │
  └─────────────────────────────────────┘

场景4: 待机 (无推理任务)
  ┌─────────────────────────────────────┐
  │ 时钟门控: 全部NPU门控               │
  │ DFS: 待机模式 (/8, 25MHz)           │
  │ 电源门控: 全部断电                   │
  │ 预期功耗: ~5%                       │
  └─────────────────────────────────────┘
```

### 6.3 低功耗技术综合收益

**源码位置：** `docs/DESIGN_REPORT.md` 第4.4.3节

| 技术 | 适用场景 | 预估功耗节省 |
|------|----------|--------------|
| 时钟门控（Core级） | NPU部分Core空闲 | 25-55% |
| 时钟门控（全NPU） | NPU完全空闲 | 60-77% |
| DFS (100MHz) | 轻推理任务 | ~49% |
| DFS (50MHz) | 极轻推理任务 | ~74% |
| 电源门控（Core级） | NPU Core长期空闲 | ~95%（该Core） |

**三级组合效果（LeNet场景）：**

```
基准功耗 (8核@200MHz): 100%

Step 1: 时钟门控6个空闲Core → 45.1%
Step 2: DFS降至50MHz → 45.1% × 0.5 ≈ 22.6%
Step 3: 电源门控6个Core (消除漏电) → 22.6% × 0.6 ≈ 13.5%

总节省: ~86.5% (仅消耗基准的13.5%)
```

### 7.1 NPU Core门控时钟生成

```verilog
// 每个NPU Core的时钟门控
module npu_core_clk_gate (
    input  wire clk_in,
    input  wire rst_n,
    input  wire core_en,      // 来自NPU_CORE_MASK
    output wire clk_out
);
    reg clk_en_latch;

    always @(negedge clk_in or negedge rst_n) begin
        if (!rst_n)
            clk_en_latch <= 1'b0;
        else
            clk_en_latch <= core_en;
    end

    assign clk_out = clk_in & clk_en_latch;
endmodule
```

### 7.3 电源门控状态机

```verilog
// 简化的电源门控控制器
typedef enum logic [2:0] {
    PG_ON,       // 正常供电
    PG_SAVE,     // 保存状态
    PG_ISOLATE,  // 使能隔离
    PG_OFF,      // 断电
    PG_RELEASE,  // 释放隔离
    PG_RESTORE,  // 恢复状态
    PG_STABLE    // 等待稳定
} pg_state_t;

always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        pg_state <= PG_ON;
    end else begin
        case (pg_state)
            PG_ON: begin
                if (pg_request_off && core_idle)
                    pg_state <= PG_SAVE;
            end
            PG_SAVE: begin
                // 保存关键寄存器到保持寄存器
                pg_state <= PG_ISOLATE;
            end
            PG_ISOLATE: begin
                // 使能隔离信号
                pg_state <= PG_OFF;
            end
            PG_OFF: begin
                // 切断Sleep FET
                if (pg_request_on)
                    pg_state <= PG_RELEASE;
            end
            PG_RELEASE: begin
                // 释放隔离, 恢复供电
                pg_state <= PG_RESTORE;
            end
            PG_RESTORE: begin
                // 从保持寄存器恢复状态
                pg_state <= PG_STABLE;
            end
            PG_STABLE: begin
                // 等待电源稳定 (~32 cycles)
                pg_state <= PG_ON;
            end
        endcase
    end
end
```

## 8. 关键知识点总结

| 知识点 | 核心公式/要点 |
|--------|--------------|
| 动态功耗 | P = alpha × C × V^2 × f |
| 静态功耗 | P = V × I_leak, 工艺越先进占比越大 |
| 时钟门控 | Latch + AND, 消除alpha, 1周期生效 |
| DFS | 线性降低f, 无毛刺切换, 主要省瞬时功耗 |
| 电源门控 | 切断V_DD, 消除P_static, 需隔离+保持 |
| 脉动阵列低功耗 | 局部通信(低C), 数据复用(低alpha), 权重驻留(低SRAM访问) |
| 三级体系 | CG(即时) → DFS(几周期) → PG(几十周期), 粒度逐级增大 |

## 10. 常见问题

**Q1: 时钟门控和数据门控有什么区别？**

时钟门控切断时钟信号，所有寄存器停止翻转。数据门控（operand isolation）在数据路径上插入门控，阻止无效数据进入组合逻辑。两者可组合使用。

**Q2: 为什么不直接降低电压来节省功耗？**

电压降低会导致电路延迟增大（delay ∝ V/(V-Vth)^alpha），可能无法满足时序约束。FPGA通常不支持动态电压调整（DVS），需要ASIC才能实现。DFS是FPGA上更实用的替代方案。

**Q3: 电源门控在FPGA上能实现吗？**

FPGA的电源域是固定的，无法像ASIC那样细粒度控制。但现代FPGA（如Xilinx UltraScale+）支持部分重配置（Partial Reconfiguration），可以实现模块级的电源管理。

**Q4: 脉动阵列的低功耗优势在什么条件下最明显？**

当矩阵维度远大于阵列规模时，数据复用率最高，SRAM访问功耗节省最显著。对于小矩阵（如1x1卷积），脉动阵列的优势会减弱。
