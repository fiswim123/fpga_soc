## 1. 卷积运算的数学定义

### 1.1 二维卷积

对于输入特征图 X、卷积核 W、输出特征图 Y，二维卷积定义为：

```
Y[n][oh][ow] = Σ_c Σ_kh Σ_kw  X[c][oh+kh][ow+kw] × W[n][c][kh][kw]

其中:
  n   = 输出通道索引 (0 .. Cout-1)
  c   = 输入通道索引 (0 .. Cin-1)
  kh  = 卷积核高度索引 (0 .. K-1)
  kw  = 卷积核宽度索引 (0 .. K-1)
  oh  = 输出高度坐标
  ow  = 输出宽度坐标
```

**直观理解**: 卷积核在输入上滑动，每个位置做"乘加"累加，生成一个输出值。

### 1.2 带 Padding 的卷积

```
输入尺寸: H_in × W_in
卷积核:   K × K
Padding:  P
Stride:   S (默认=1)

输出尺寸:
  H_out = (H_in + 2P - K) / S + 1
  W_out = (W_in + 2P - K) / S + 1
```

**示例 — Conv1 (CIFAR-10)**:
```
H_in = 32, W_in = 32, K = 5, P = 2, S = 1
H_out = (32 + 2×2 - 5) / 1 + 1 = 32
W_out = (32 + 2×2 - 5) / 1 + 1 = 32

输入:  32×32×3   (CIFAR-10 RGB)
输出:  32×32×32  (32个输出通道)
```

## 设计视角：为什么这样设计？

### 动机：为什么需要 im2col 变换？

卷积运算本质上是"滑窗乘加"，而脉动阵列天然适合矩阵乘法。
直接在硬件上实现滑窗逻辑需要复杂的地址生成和数据路由。

```
  问题: 卷积 vs 矩阵乘法

  卷积 (滑窗):
    for each output position (oh, ow):
      for each kernel element (kh, kw):
        Y[oh][ow] += X[oh+kh][ow+kw] * W[kh][kw]

    特点: 输入数据被多次访问 (重叠窗口)
           地址计算复杂 (需要加法和边界检查)

  矩阵乘法:
    Y = A × B
    Y[i][j] = Σ_k A[i][k] * B[k][j]

    特点: 规则的行列访问
           脉动阵列天然支持

  im2col 的作用: 将卷积的"滑窗访问"转化为矩阵的"行列访问"
```

### 为什么选择 INT8 量化？

```
  精度 vs 硬件成本权衡:

  ┌──────────┬──────────┬──────────┬──────────┐
  │ 数据类型  │ 位宽     │ 乘法器面积│ 精度损失  │
  ├──────────┼──────────┼──────────┼──────────┤
  │ FP32     │ 32 bit   │ 大 (~5K) │ 无       │
  │ FP16     │ 16 bit   │ 中 (~2K) │ 小       │
  │ INT8     │ 8 bit    │ 小 (~200)│ 可接受   │
  │ INT4     │ 4 bit    │ 极小     │ 较大     │
  └──────────┴──────────┴──────────┴──────────┘

  INT8 优势:
    - 乘法器面积仅为 FP32 的 1/25
    - 1280 个 PE 的 INT8 乘法器可放入 FPGA
    - CIFAR-10 分类精度损失 < 1%
    - 饱和处理简单 (clamp to [-128, 127])
```

### 设计约束总结

```
CNN 硬件映射约束:

  约束 1: im2col 矩阵必须能放入片上 RAM
    Conv1: 1024×75×8b = 75KB
    Conv2: 256×800×8b = 200KB

  约束 2: 权重矩阵必须能放入片上 RAM
    Conv1: 75×32×8b = 2.4KB
    Conv2: 800×64×8b = 50KB

  约束 3: 量化精度满足分类要求
    INT8 量化后准确率 > 90% (CIFAR-10)

  约束 4: 计算量在硬件吞吐范围内
    Conv1: 1024×75×32 = 2.46M MAC
    Conv2: 256×800×64 = 13.1M MAC
```

## 设计视角：如何从零开始设计？

### 步骤 1: 分析网络计算图

```
  输入: CNN 模型定义 (层类型、参数、连接关系)

  CIFAR-10 计算图:
    Conv1(5x5, 3→32) → ReLU → MaxPool(2x2)
    → Conv2(5x5, 32→64) → ReLU → MaxPool(2x2)
    → GAP → FC(64→10) → argmax

  每层提取:
    - 输入尺寸 (H, W, Cin)
    - 卷积核尺寸 (K, Cout)
    - Padding, Stride
    - 输出尺寸
```

### 步骤 2: 计算 im2col 矩阵尺寸

```
  对于每一层卷积:
    行数 = 输出位置数 = H_out × W_out
    列数 = K² × Cin

  Conv1:
    行数 = 32×32 = 1024
    列数 = 5×5×3 = 75
    矩阵大小 = 1024×75 = 76,800 元素

  Conv2:
    行数 = 16×16 = 256
    列数 = 5×5×32 = 800
    矩阵大小 = 256×800 = 204,800 元素
```

### 步骤 3: 设计 im2col 地址计算逻辑

```
  核心: 从 (row, k_idx) 计算源数据地址

  row → (oh, ow):
    oh = row / W_out
    ow = row % W_out

  k_idx → (ch, kh, kw):
    ch = k_idx / (K × K)
    kh = (k_idx % (K × K)) / K
    kw = k_idx % K

  源地址:
    ih = oh + kh - pad
    iw = ow + kw - pad
    if (ih < 0 || ih >= H || iw < 0 || iw >= W)
      data = 0  // padding
    else
      data = input[ih][iw][ch]
```

### 步骤 4: 设计量化方案

```
  训练阶段: FP32 权重和激活
    ↓
  量化阶段:
    scale = (max - min) / 255
    q_weight = round(fp_weight / scale)
    ↓
  推理阶段: INT8 乘累加
    result_i32 = Σ (a_i8 × w_i8)
    ↓
  反量化:
    out_shift 控制截断位置
    result_i8 = clamp(result_i32 >> out_shift, -128, 127)
```

### 步骤 5: 映射到脉动阵列

```
  矩阵乘法 Y = A × B 的分块:

  A: M×K (im2col)    B: K×N (权重)    C: M×N (输出)

  阵列尺寸: 40×32 PE
    行方向: 每次处理 40 行 (tile)
    列方向: 每次处理 32 列 (pass)

  Conv1: M=1024, K=75, N=32
    Tile 数 = ceil(1024/40) = 26
    Pass 数 = ceil(32/32) = 1
    总计算轮次 = 26 × 1 = 26

  Conv2: M=256, K=800, N=64
    Tile 数 = ceil(256/40) = 7
    Pass 数 = ceil(64/32) = 2
    总计算轮次 = 7 × 2 = 14
```

## 3. im2col 变换原理

### 3.1 为什么需要 im2col?

脉动阵列 (Systolic Array) 天然适合矩阵乘法 `Y = A × B`，但卷积是"滑窗乘加"。

**解决方案**: 将输入特征图重排为矩阵，使卷积变成矩阵乘法。

### 3.2 im2col 变换示意

以 3x3 输入、2x2 卷积核、1x1 输出为例:

```
原始输入 X (3×3):          im2col 展开 A (4×4):
┌───┬───┬───┐              ┌───┬───┬───┬───┐
│ a │ b │ c │              │ a │ b │ d │ e │  ← 位置(0,0)
├───┼───┼───┤              ├───┼───┼───┼───┤
│ d │ e │ f │              │ b │ c │ e │ f │  ← 位置(0,1)
├───┼───┼───┤              ├───┼───┼───┼───┤
│ g │ h │ i │              │ d │ e │ g │ h │  ← 位置(1,0)
└───┴───┴───┘              ├───┼───┼───┼───┤
                           │ e │ f │ h │ i │  ← 位置(1,1)
                           └───┴───┴───┴───┘

卷积核 W (2×2):            展开为 B (4×1):
┌───┬───┐                  ┌───┐
│ w0│ w1│                  │ w0│
├───┼───┤                  ├───┤
│ w2│ w3│                  │ w1│
└───┴───┘                  ├───┤
                           │ w2│
                           ├───┤
                           │ w3│
                           └───┘

矩阵乘法: Y = A × B
┌───┬───┬───┬───┐   ┌───┐   ┌─────────────────┐
│ a │ b │ d │ e │   │ w0│   │ aw0+bw1+dw2+ew3 │  ← Y[0,0]
├───┼───┼───┼───┤   ├───┤   ├─────────────────┤
│ b │ c │ e │ f │ × │ w1│ = │ bw0+cw1+ew2+fw3 │  ← Y[0,1]
├───┼───┼───┼───┤   ├───┤   ├─────────────────┤
│ d │ e │ g │ h │   │ w2│   │ dw0+ew1+gw2+hw3 │  ← Y[1,0]
├───┼───┼───┼───┤   ├───┤   ├─────────────────┤
│ e │ f │ h │ i │   │ w3│   │ ew0+fw1+hw2+iw3 │  ← Y[1,1]
└───┴───┴───┴───┘   └───┘   └─────────────────┘
```

**这就是卷积的矩阵乘法形式!**

### 3.3 多通道 im2col

对于 Conv1 (3通道输入, 5x5核, 32通道输出):

```
A 矩阵 (im2col 展开):           B 矩阵 (权重):

┌─────────────────────┐          ┌─────────────────────┐

│                     │          │ ...                 │

│ 的 5×5 邻域展开     │          │                     │
│ ...                 │          │ (每行256bit = 32ch×8b)│

│                     │          │                     │

矩阵乘法: Y = A × B
  输出 Y: 1024行 × 32列 (INT32累加, 量化后INT8)
```

### 3.4 硬件实现: 组合逻辑 im2col

在我们的 NPU 中，im2col 是**纯组合逻辑**，零延迟完成地址计算和数据提取。

**源码位置**: `src/npu/dmac_im2col_stream.sv`

```systemverilog
// 文件: src/npu/dmac_im2col_stream.sv, line 109-152
function automatic signed [7:0] get_lane_data(input int lane);
    int row, oh, ow, ch, kh, kw, ih, iw;
    logic signed [7:0] val;
    begin
        row = row_base + lane;              // 全局输出行索引
        oh  = row / cfg_in_w;               // 输出 H 坐标
        ow  = row % cfg_in_w;               // 输出 W 坐标
        ch  = k_idx / (cfg_kernel * cfg_kernel);  // 输入通道
        kh  = (k_idx % (cfg_kernel * cfg_kernel)) / cfg_kernel;
        kw  = k_idx % cfg_kernel;
        ih  = oh + kh - cfg_pad;            // 输入 H (含 padding)
        iw  = ow + kw - cfg_pad;            // 输入 W (含 padding)

        // 边界检查 (padding = 0)
        if (row >= cfg_in_w * cfg_in_h || ch >= cfg_in_ch ||
            ih < 0 || ih >= cfg_in_h || iw < 0 || iw >= cfg_in_w)
            val = 8'sd0;
        else if (layer_sel == 1'b0)
            val = image_pixel_channel(image_buf[ih * cfg_in_w + iw], ch);
        else
            val = pool_buf[ih * cfg_in_w + iw][255 - ch*8 -: 8];
        get_lane_data = val;
    end
endfunction
```

**关键设计**:
- 每次调用返回一个 lane (一个 im2col 列) 的数据
- 地址计算全部用整数除法/取模完成
- padding 区域直接返回 0
- Layer 1 从 `image_buf` 读取, Layer 2 从 `pool_buf` 读取

## 4. INT8 量化

### 4.1 为什么需要量化?

神经网络的权重和激活值通常是浮点数 (FP32)，但硬件实现浮点运算成本高。

**解决方案**: 将浮点数量化为定点整数 (INT8)，用整数乘加替代浮点运算。

### 4.2 量化公式

```
浮点值 → INT8:
  q = round(x / scale)  + zero_point
  scale = (x_max - x_min) / 255

INT8 → 浮点值:
  x = (q - zero_point) × scale
```

在我们的 NPU 中，简化为:
```
INT8 范围: -128 ~ +127
权重: 直接用 INT8 存储 (训练时已量化)
激活: ReLU 后截断到 INT8 范围
```

### 4.3 量化截断 (硬件实现)

**源码位置**: `src/npu/mm_systolic_4x4.sv`

```systemverilog
// 文件: src/npu/mm_systolic_4x4.sv, line 134
pe_res_i8[m][n] = {pe_res[m][n][31], pe_res[m][n][out_shift +: 7]};

// out_shift 控制截断位置:
//   Layer 1: out_shift = 7  (右移7位)
//   Layer 2: out_shift = 8  (右移8位)
```

**截断示意**:
```
32bit 累加结果: [31] [30:8] [7:0]
                 符号  数据   低位

取高8位:
  INT8 = {bit[31], bit[out_shift+6 : out_shift]}
  相当于: result >> out_shift, 然后取低8位

out_shift=7: 有效范围约 ±16384 (2^14)
out_shift=8: 有效范围约 ±32768 (2^15)
```

### 4.4 饱和处理

量化截断后需要饱和到 INT8 范围:

```systemverilog
// 文件: src/npu/mm_systolic_4x4.sv, line 152-162
function automatic signed [7:0] sat_i8(input logic signed [31:0] vin);
    begin
        if (vin > 32'sd127) begin
            sat_i8 = 8'sd127;
        end else if (vin < -32'sd128) begin
            sat_i8 = -8'sd128;
        end else begin
            sat_i8 = vin[7:0];
        end
    end
endfunction
```

**示例**:
```
输入 vin = 200  → sat_i8 = 127  (上溢饱和)
输入 vin = -200 → sat_i8 = -128 (下溢饱和)
输入 vin = 50   → sat_i8 = 50   (正常范围)
```

## 5. 矩阵乘法与脉动阵列

### 5.1 矩阵乘法定义

```
C = A × B

其中:
  A: M × K 矩阵 (im2col 展开的输入)
  B: K × N 矩阵 (权重)
  C: M × N 矩阵 (输出)

C[i][j] = Σ_{k=0}^{K-1} A[i][k] × B[k][j]
```

### 5.2 脉动阵列执行矩阵乘法

```
时间步 0:          时间步 1:          时间步 2:
A[0][0] → PE      A[0][1] → PE      A[0][2] → PE
B[0][0] ↓ PE      B[1][0] ↓ PE      B[2][0] ↓ PE

PE 累加:
  cycle 0: acc = A[0][0] × B[0][0]
  cycle 1: acc += A[0][1] × B[1][0]
  cycle 2: acc += A[0][2] × B[2][0]
  ...经过 K 个周期后得到 C[0][0]
```

### 5.3 我们的 NPU 阵列规模

```
MAC 阵列: 40行 × 32列 = 1280 个 PE
  = 10 × 8 个 4×4 子阵列
  = 每次计算 40 个输出行 × 32 个输出通道

Conv1:
  A: 1024行 × 75列  → 分成 26 个 tile (每 tile 40 行)
  B: 75行 × 32列
  每 tile 计算: 40×32 = 1280 个输出值

Conv2:
  A: 256行 × 800列  → 分成 7 个 tile
  B: 800行 × 64列   → 分成 2 个 pass (每 pass 32 列)
```

## 6. 关键知识点总结

### 6.1 核心概念

| 概念 | 含义 | 硬件对应 |
|------|------|----------|
| im2col | 卷积→矩阵乘法 | dmac_im2col_stream |
| INT8量化 | 浮点→定点整数 | sat_i8 函数 |
| 脉动阵列 | 并行MAC单元网格 | mm_systolic_4x4 |
| 权重驻留 | 权重固定在PE中 | Weight Stationary |
| Tile化 | 分块复用权重 | tile 循环控制 |

### 6.2 数据流路径

```
image_data.dat
     │
     ▼
image_buf (24bit/pixel)
     │
     │ im2col (组合逻辑)
     ▼
image_sa_ram (320bit/行)
     │
     │ 每周期读1行
     ▼
MAC 阵列 (40×32 PE)
     │
     │ INT8 累加
     ▼
result_ram (256bit/行)
     │
     │ MaxPool
     ▼
pool_ram → pool_buf (供下一层)
```

### 6.3 存储资源

| 存储 | 位宽 | 深度 | 用途 |
|------|------|------|------|
| image_buf | 24b | 1024 | 原始图像 |
| image_sa_ram | 320b | 5600 | im2col矩阵 |
| weight_buf | 256b | 75 | Conv1权重 |
| weight2_buf | 512b | 800 | Conv2权重 |
| result_ram | 256b | 1024 | 卷积结果 |
| pool_ram | 256b | 256 | 池化结果 |
