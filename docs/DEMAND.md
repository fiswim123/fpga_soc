# DEMAND — 异构处理器设计需求文档

> 基于 第九届中国研究生创"芯"大赛 飞腾企业命题 赛题三：智核融合·低耗强算
> 整合官方赛题说明 + 全部论坛讨论答疑（23帖）

---

## 1. 项目概述

设计一款 **32位低功耗RISC处理器 + 32位NPU** 的异构SoC，通过 **AXI总线** 实现CPU与NPU的高效协同通信，面向物联网、边缘AI推理场景（智能家居人脸识别、工业缺陷检测、可穿戴ECG分析等）。

**核心目标**：CPU负责逻辑控制与任务调度，NPU负责矩阵/卷积并行计算，实现低功耗下的高能效AI推理。

---

## 2. 处理器选型

### 2.1 CPU（二选一）

| 方案 | 来源 | AXI接口 | 说明 |
|------|------|---------|------|
| ARM Cortex-M0 | ARM官方评估版 | — | 32位低功耗核 |
| **PicoRV32**（推荐） | [github.com/YosysHQ/picorv32](https://github.com/YosysHQ/picorv32) | 自带AXI4 | RISC-V RV32 |
| Darkriscv | [github.com/darklife/darkriscv](https://github.com/darklife/darkriscv) | 无AXI（需自行添加，参考PicoRV32） | RISC-V RV32 |

- RISC-V核**允许自定义修改**，但需遵循RISC-V规范并协调软硬件适配。
- CPU必须是**指定的三款之一**，不可使用其他处理器（如飞腾D2000），也不可用PYNQ/Vitis等平台替代CPU。

### 2.2 NPU

- **必须完全自行设计**，不可使用第三方NPU IP。
- 基础架构：**4×4 脉动阵列（Systolic Array）**，允许扩展为更大规模（如16×16）或多核并行。
- NPU**不限单核**，可使用多个4×4阵列组成更大算力集群。
- 数据通路字宽：**32位**（ALU、寄存器等核心计算单元为32位宽）。
- 计算精度考核指标：**INT8**（可额外支持FP32等，但评分只看INT8 TOPS）。

## 3. 总线与互连架构

### 3.1 AXI接口要求（强制）

| 接口类型 | 用途 | 说明 |
|----------|------|------|
| **AXI-Lite** | CPU ↔ NPU 控制面（CSR寄存器访问） | 单拍传输，memory-mapped slave |
| **AXI-Full** | CPU/NPU ↔ 存储器 数据面 | Burst传输，支持地址递增 |

**关键约束**：
- CPU必须通过 **AXI-Lite memory-mapped slave** 访问NPU的CSR寄存器，**不允许使用PCPI自定义指令**等方式替代。
- 数据面AXI-Full位宽**不强制32bit**，可采用64bit或128bit以提升带宽（总线利用率考核与位宽无关）。

### 3.2 互连拓扑（加分项）

- 基础：CPU与NPU通过AXI总线直连。
- 加分：**AXI共享总线互连（AXI Interconnect）**，支持多个Master/Slave并行访问，提升系统可用带宽。

## 4. 性能指标

### 4.1 基础指标（必须达到）

| 指标 | 要求 |
|------|------|
| RTL仿真频率 | **200 MHz** |
| NPU峰值算力 | **≥ 0.5 TOPS@INT8**（理论峰值） |
| 总线带宽利用率 | **≥ 60%**（Burst传输场景） |
| 代码覆盖率 | **≥ 95%** |

### 4.2 优化指标（加分）

| 指标 | 目标 |
|------|------|
| NPU算力 | 接近或超过 **1 TOPS@INT8**（峰值） |
| 总线带宽利用率 | **≥ 80%**（Burst传输场景） |

### 4.3 算力计算说明

- 算力按硬件**理论峰值**计算：`TOPS = MAC数量 × 2(乘+加) × 频率 / 1e12`
- 单一4×4阵列@200MHz = 16 × 2 × 200MHz = **6.4 GOPS**，远达不到0.5 TOPS
- 需通过以下方式扩展：阵列规模扩大（如16×16）、多阵列并行、时间复用、优化MAC单元
- 1 TOPS指标面向**可扩展架构**的理论峰值，非单一4×4阵列可达
- **考核基准是INT8**，不通过FP32换算，需聚焦INT8运算优化
- FPGA仅需验证功能正确性，算力以ASIC为目标评估

## 5. 低功耗设计要求

### 5.1 基础要求

- **时钟门控（Clock Gating）**：NPU空闲时关闭脉动阵列时钟，降低静态功耗。

### 5.2 优化加分项

| 技术 | 说明 |
|------|------|
| **动态频率调整（DFS）** | 根据任务负载动态调整NPU工作频率 |
| 电源门控（Power Gating） | 进一步降低待机功耗 |
| 多电压域设计 | 不同模块采用不同电压域 |
| 其他创新低功耗方法 | 自行探索 |

- "低功耗"是**设计策略**概念，非绝对功耗数值要求；重点在于设计中有低功耗技术手段。

## 6. 功能要求

### 6.1 基础功能

| 功能 | 描述 |
|------|------|
| AXI Burst传输 | 地址递增模式，数据正确无误 |
| CPU+NPU协同任务 | NPU执行矩阵/卷积运算，CPU处理逻辑控制与调度 |
| NPU CSR配置 | CPU通过AXI-Lite访问NPU控制寄存器 |

### 6.2 硬件加速扩展（加分项）

| 功能 | 描述 |
|------|------|
| **动态可调脉动阵列** | 硬件支持通过配置改变PE间连接方式或数据流路径，实现不同规模的子阵列映射 |
| **DMA控制器** | 集成DMA实现数据高效搬运，减少CPU参与数据传输的开销 |

### 6.3 AI推理

- 建议在**硬件层面**实现卷积层、全连接层、池化层（非纯软件im2col映射）。
- 可根据资源约束选择实现方式，需在文档中说明选择依据。

## 7. 验证与测试

### 7.1 仿真平台

| 项目 | 要求 |
|------|------|
| 仿真工具 | **VS Code + iverilog**（极力推荐）；可用Vivado但iverilog优先 |
| 代码覆盖率 | **≥ 95%** |
| IP使用 | Vivado AXI Interconnect / SRAM IP可用于仿真 |
| 存储模型 | CPU自带存储模型及接口；可自行选择Verilog存储模型 |

### 7.2 功能验证

- 完成AXI Busrt传输正确性测试（地址递增模式）
- CPU与NPU协同任务测试
- 压力测试与边界条件测试

### 7.3 性能测试

| 测试项 | 方法 |
|------|------|
| AI推理 | 标准测试集（**MNIST**、**CIFAR-10**等），记录推理时间和准确率 |
| 推荐网络 | MNIST → LeNet；CIFAR-10 → ResNet（允许自选网络，需公开模型细节） |
| 总线利用率 | 测量Burst传输场景 |
| NPU算力 | 理论峰值计算 + 可选的实测算力（如ResNet-18） |
| 推理方式 | 逐帧或批量均可，批量能更好利用脉动阵列流水线 |

### 7.4 低功耗测试

- 不同工作负载下测量功耗，验证低功耗设计有效性
- 评估DFS等功能对功耗的影响

### 7.5 FPGA验证（加分项，+10分）

- FPGA可做**小规模NPU验证**（如16个4×4阵列）
- FPGA仅需验证功能正确性，不要求达到0.5 TOPS（受开发板资源限制）
- 算力以ASIC为目标评估，FPGA聚焦功能验证

## 8. 面积与存储资源约束

- CPU+NPU整体面积：**无硬性限制**，但需在技术报告中说明面积与性能的权衡，避免因面积过大导致功耗或成本超标
- 全局Buffer / 片上缓存：**允许且推荐**，可提升数据复用效率、降低访存开销
- NPU局部Buffer / 系统级全局Buffer：**无明确容量上限**，建议：
  - 局部Buffer与NPU核心算力匹配
  - 全局Buffer通过代价模型动态调整容量
- 面积与存储配置：**不直接纳入评分**，但建议在报告中量化其对性能和能效的提升

## 12. FAQ 关键澄清汇总

| # | 问题 | 结论 |
|---|------|------|
| 1 | 4×4阵列能到0.5TOPS吗？ | 不能，需扩展阵列或多核并行，指标面向可扩展架构峰值 |
| 2 | 32位NPU含义？ | 数据通路字宽（ALU、寄存器位宽），数据面AXI位宽不限 |
| 3 | 必须用iverilog吗？ | 极力推荐，但Vivado也可 |
| 4 | 能用PCPI代替AXI-Lite吗？ | **不可以**，必须AXI-Lite memory-mapped访问CSR |
| 5 | NPU必须单核吗？ | 不限，可用多核/多阵列 |
| 6 | RISC-V核能改吗？ | 可以，需遵循RISC-V规范 |
| 7 | 频率必须200MHz吗？ | 是，符合嵌入式处理器典型频率（100-300MHz） |
| 8 | 性能时间怎么测？ | 看波形分析，不用CPU打印时间 |
| 9 | FPGA要达到0.5TOPS吗？ | 不需要，FPGA只验证功能，算力以ASIC评估 |
| 10 | Buffer大小有限制吗？ | 无硬性上限，需自行权衡面积/功耗 |
